How the Agents Work

1. Research agentagent/research.py starts from data/apps.json and fills missing fields into output/results.json.
2. Fast pathIt checks Composio toolkit metadata first. When a toolkit exists, it records auth, tool count, MCP claim, buildability, and Composio evidence.
3. Web fallbackIf no toolkit is found, it searches for API docs, extracts page text, and asks the LLM to structure the seven research fields.
4. Pattern analysisagent/pattern.py groups results by auth type, access model, blockers, easy wins, and low-confidence rows.
5. Verificationagent/verify.py samples apps into templates, scores human verdicts, and compares pass 1 vs pass 2.
6. Report layerThis HTML summarizes the raw outputs and highlights what still requires official documentation checks.

What the Output Folder Means

The output folder has claim data, pattern summaries, manual review templates, scored verification passes, and this report. The most important distinction is that results.json contains claims while the verification_* files contain checks against official docs.
